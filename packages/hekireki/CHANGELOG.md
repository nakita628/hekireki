# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.10.0] - 2026-07-17

### Added

- Active Record (Ruby on Rails) model generator: `hekireki-activerecord` — associations (`belongs_to`, `has_one`, `has_many`, `has_and_belongs_to_many`), enums, composite primary keys, `@map`/`@@map` support
- Eloquent (Laravel / PHP) model generator: `hekireki-eloquent` — relations, `$fillable`, `$casts`, string-backed PHP enums, timestamp constants, `namespace` config
- UUID v7 and ULID primary-key generation across all ORM generators: SQLAlchemy (`uuid6` / `python-ulid`), GORM (`google/uuid` / `oklog/ulid` fed by `crypto/rand`), Sea-ORM (`uuid` / `ulid` crates), Ecto (`autogenerate: [version: 7]` on Ecto 3.14+ / `Ecto.ULID` via `ecto_ulid_next`), Active Record (`SecureRandom.uuid_v7` on Ruby 3.4+ / `ULID.generate`), Eloquent (`HasUuids` / `HasVersion4Uuids` / `HasUlids` traits)
- UUID v4 client-side generation for ORMs that previously emitted none (SQLAlchemy, GORM, Sea-ORM, Active Record, Eloquent)
- Ecto: `source:` mapping for `@map`-ped autogenerated primary keys

### Fixed

- Drizzle emitted UUID v4 (`crypto.randomUUID()`) for `@default(uuid(7))`; it now emits `v7()` from the `uuid` package
